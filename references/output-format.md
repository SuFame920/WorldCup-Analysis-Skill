# Output Format (输出格式)

Produce the report **in Chinese**, in exactly these three blocks, in this order. Keep the
reasoning visible — every pick carries its own sub-analysis, and nothing is asserted that the
evidence doesn't support.

## The template (authoritative)

```
一、整体方向分析
<2–4 段你的总体研判>。明确点出支撑判断的关键信息源,而不是空谈:
  · 评级/实力差(Channel A)
  · 近况与底层数据 xG(Channel D)
  · 阵容/伤病/停赛(Channel F)——区分"已确认"与"预计"
  · 对位与默契(Channel C/E)——天赋天花板要扣默契折扣
  · 盘口隐含概率(Channel G)——以及我和市场分歧在哪
给出方向性结论:谁占优、占多大优、平局风险多高、比赛大体走向(开放对攻 / 闷 / 一边倒)。
**爆冷概率:____%(或 低/中/高)** + 最可能引爆的 1–2 个机制。
  注意:这**不是**弱队"平+胜"概率的简单相加,是独立整体研判(算法见 reasoning-and-synthesis 专节)。

二、分项预测(每项附"子分析",比整体更细;说人话、给术语白话解释;**给完预测就站住,不准兜底;
   同类内置信度高的排最前**)

【胜平负】(至多 2 个)
  预测:____ (高置信在前)   置信度:NN/100 (/ 第二选 MM/100)
  子分析:从比分矩阵汇总得到的 H/平/A 概率 + 主要驱动机制 + 为什么是这个而非相邻选项。

【让球胜负】(至多 2 个;**体彩整数让球线先联网查到**)
  当前盘口(体彩):____ 让/受让 ____ 球(整数,如 -1 / +1;联网确认;查不到注明"线未确认,定性判断")
  预测:____(让球后的 胜/平/负,主队视角)   置信度:NN/100
  子分析:把整数让球套到比分矩阵(把受让方加上让球数)再汇总让球后胜平负的概率。
  **不要用 -0.25 / -0.5 / 1.25 这类亚洲盘四分之一/半球**;一律按体彩整数盘说"让 N 球后谁胜/平/负"。

【比分】(至多 3 个;置信度越高个数越少,拿得准给 1–2 个)
  预测:____ (/ ____ / ____)   每个后缀 置信度:NN/100,高的在前
  子分析:矩阵概率最高的几个比分。**必须与胜平负方向一致**:押主胜→只列主胜比分;押双选(胜+平)→
  只列主胜或平局比分,绝不列客胜比分。头部明显领先→给 1–2 个;多个接近→给 3 个。

【进球数 / 大小球】(至多 2 个;大小球线先联网查到)
  当前盘口:大小 ____ 球(联网确认)
  预测:**总进球 ____ 球 (/ ____ 球)** —— 写**具体进球总数**,不写"3+/大球"   置信度:NN/100
  子分析:矩阵按总进球求和 + 两队 xG 与比赛状态(必胜则开放/无关紧要则保守)。
  **给的进球数必须与比分一致**(2-1=3 球、3-1=4 球):你列的比分总和要正好落在这些进球数上。

【半全场】(至多 2 个;方差最大,通常 1 个)
  记法:[半场][全场],主队(列前队)视角,各取 胜/平/负,共 9 种(胜胜/胜平/胜负/平胜/平平/
        平负/负胜/负平/负负);负胜=逆转、胜负=被翻盘,概率最低。
  预测:____ (/ ____)   置信度:NN/100
  子分析:基于上下半场各自进球速率的联合判断(别只把全场结果一拆为二)。**全场段必须与胜平负一致**
  (押主胜→全场段是"胜")。坦白说明这是最不稳的一项。

三、预测总结
  一眼看完的结论清单,直接排布(见下方示例的"总结"块);每行带 `置信度:NN/100`;
  **末尾必含一行"爆冷概率"**。
```

## 置信度 → 个数 + 0–100 分值(硬规则,不是装饰)

- **置信度用 0–100 分值**,每个预测后缀 `置信度:NN/100`(不再用"高/中/低")。参考映射:
  75–100=很有把握、55–74=偏向明显、40–54=略微倾向/接近五五、<40=只作参考。分值对应该玩法
  分布的集中度(越尖越高)。
- **个数由分布集中度决定,且有硬上限**(不确定可加到上限,**不得超过**):
  - 胜平负 ≤ 2 · 让球 ≤ 2 · 比分 ≤ 3 · 大小球 ≤ 2 · 半全场 ≤ 2。
  - 分布越尖给越少(某结果主导→1 个);越平给越多(到上限为止)。
- **同类内按置信度从高到低排序**,第一个就是你最看好的。

## 说人话 + 果断 + 不矛盾(硬规则,用户定制 ③④⑤)

- **说人话.** 面向没赌过球的小白。每个专业词第一次出现给一句白话:让球 =「赢几个球才算赢」、
  大小球 =「两队共进几球」、半全场 =「半场和全场各算一次」、1X =「押主队赢或平的保底」、
  xG =「机会质量」、Elo =「实力评分」。能用大白话就别用术语。
- **果断,不兜底.** 给完每项预测后,**禁止**再写"但 X 也很活 / 也可能 / 不排除……"这类自我拆台。
  不确定性只走三条正规渠道:① 选项个数(越不确定给越多,到上限)② 0–100 置信度分值 ③ 爆冷概率。
- **「论证反方」是内部步骤,不是输出措辞.** 推理时一定要想对手赢的剧本(见 reasoning-and-synthesis
  的 *Argue the other side*),但它的结论**流向"爆冷概率"和选项个数**,而不是写成输出里的犹豫。
- **跨玩法严禁矛盾(硬规则,写之前逐项对一遍).** 五项都出自同一张比分矩阵,**结论必须互相对得上**:
  1. **胜平负方向锁死比分、半全场、让球的方向。** 押**主胜** → 比分**只列主胜比分**(禁止平局比分、
     禁止客胜比分);半全场**全场段=胜**;体彩整数让球后的结果同向。押**双选(胜+平)** → 比分只在
     主胜/平局里取,**绝不出现客胜比分**。这是用户的硬要求:"说了挪威胜,就不会出现塞内加尔 2-1 挪威"。
  2. **大小球的进球数 = 比分总和。** 你列的比分总进球必须正好等于你给的大小球进球数(押"3 球/4 球"
     →比分用 2-1、3-1、3-0 这类;不允许列一个 1-1=2 球却押 3 球)。
  3. 其余:让球大胜 ⇎ 平局倾向;"两队都进球=是" ⇎ 某队零封的头号比分;爆冷概率 ⇎ 1X2。
  有任何一处对不上,就改到完全一致为止——**这是算术,不是态度**。

## 必含项 + 锦上添花

- **必含:爆冷概率**(整体方向里一行 + 总结里一行),含引爆机制。
- **必含:** 每个预测后缀 `置信度:NN/100`(0–100 分值,同类内高分在前)。
- **建议:** 一个 **关键变量(最可能改变结果的 2–3 件事)** 小块。
- **建议:** 一行 **盘口 vs 我的判断**——我和市场分歧最大处,以及凭什么。

## 示例(基于真实赛前背景;比分/盘口数值为演示,实战需联网确认)

> 演示赛事:2026 世界杯 F 组,突尼斯 vs 日本(6/21,蒙特雷)。真实背景:首轮突尼斯 1–5 负瑞典、
> 防线崩盘,近 5 场 1 胜 1 平 3 负、进 2 失 11;日本 2–2 战平荷兰、卡马达(Kamada)88 分钟绝平,
> 创造力核心久保(Kubo)/堂安(Doan)/卡马达,中场田中(Tanaka)+佐野(Sano)双后腰。两队均需分数。
> **以下让球线/大小球线为示例占位,真实运行时由 Channel G 联网取实时盘口。**

```
一、整体方向分析
日本占据明显但非压倒性的优势。实力评级、底层数据、对位三条线一致指向日本:首轮对荷兰能踢出
2–2 且 xG 不落下风,说明其控球—边路推进体系成型;久保、堂安、卡马达的创造力对突尼斯刚刚被瑞典
打穿的防线是直接威胁(突尼斯首轮失 5 球、近 5 场失 11,防守组织是结构性问题而非偶然)。突尼斯
的现实指望是低位密集 + 定位球(唯一进球来自雷基克头槌),但这需要中场顶住日本的控球节奏,而这恰
是他们最吃紧的地方。默契与状态上日本更稳(主力框架延续),突尼斯刚换帅压力下需要重组。
分歧提示:盘口若给日本较深的让步,需警惕——世界杯单场方差大,突尼斯摆大巴 + 日本临门效率波动
会压低日本的过盘概率,因此我对"让球"比对"胜平负"更谨慎。
整体走向:日本控球主导、突尼斯被动防反 + 拼定位球;日本进球概率高,但若久保/卡马达任一不首发,
威胁度明显下降。
爆冷概率:约 20%。引爆点:① 突尼斯摆全场铁桶 + 门将神勇,把日本拖成 0–0 或被偷一球反杀;
② 久保或卡马达意外缺阵,日本破密集能力骤降。(这不是"突尼斯平+胜"的简单加总,是把催化剂算进去的整体判断。)

> 注:此示例以**主队突尼斯**为列前队;胜平负预测"日本胜"=突尼斯负,故下面所有比分都是突尼斯输的比分、
> 半全场全场段是"负",彼此不矛盾。

二、分项预测

【胜平负】(1 个) 预测:日本胜  置信度:66/100
  子分析:矩阵汇总日本胜约六成、平约两成出头、突尼斯胜两成以内。驱动:体系与创造力差 + 突尼斯
  防守结构性漏洞。头部明显偏向日本所以只给 1 个;但不上调到"大胜",因突尼斯低位防守会压低净胜球。

【让球胜负】(1 个)
  当前盘口(体彩):日本 让 1 球(整数盘;示例,需联网确认)
  预测:让 1 球后 **平局**(即日本恰好净胜 1 球)  置信度:48/100
  子分析:体彩"日本 -1"=日本赢 2 球及以上才算赢、净胜 1 球算走盘(平)、其它算输。突尼斯摆大巴
  把比赛压成小比分,日本最可能的赢法是 1–0/2–1(净胜 1),落在"让 1 球后平局"格。与"日本胜"不矛盾:
  日本仍赢球,只是赢的幅度卡在让球线上。

【比分】(3 个) 预测:1–0 日本 / 2–1 日本 / 2–0 日本  置信度:1–0=24/100 · 2–1=20/100 · 2–0=17/100
  子分析:三个都是日本小胜(与"日本胜"一致),按概率从高到低排。头部较分散(突尼斯低位让"日本
  小胜"成为一簇接近结果),故给到上限 3 个。总进球分别是 1、3、2 球——见下面大小球需自洽。

【进球数 / 大小球】(2 个)
  当前盘口:大小 2.5 球(示例,需联网确认)
  预测:**总进球 2 球 / 1 球**  置信度:2 球=30/100 · 1 球=26/100
  子分析:日本进攻强但突尼斯主动压缩空间、自身进攻贫弱(近 5 场进 2),矩阵质量集中在 1–2 球。
  自洽:比分领衔的 1–0(1 球)、2–0(2 球)正落在这里;2–1(3 球)是同方向的次选,不影响主判。

【半全场】(1 个) 预测:平负(半场平、全场日本胜;主队突尼斯视角)  置信度:38/100
  子分析:突尼斯开场先求稳、日本破密集需要时间,上半场 0–0、下半场日本打开概率最高。全场段"负"
  与胜平负"日本胜"一致。坦白:半全场方差最大,仅作低权重参考。

三、预测总结
  · 胜平负:日本胜(66)
  · 让球(体彩 日本 -1 示例):让 1 球后平局/日本净胜 1 球(48)——以实时盘口为准
  · 比分:1–0 / 2–1 / 2–0 日本(24 / 20 / 17)
  · 大小球(2.5 示例):2 球 / 1 球(30 / 26)
  · 半全场:平负(半场平、全场日本胜)(38)
  · 关键变量:久保&卡马达是否首发;突尼斯是否仍摆大巴;日本是否早进球打开局面
  · 盘口 vs 我:体彩让 1 球我偏向"日本只净胜 1 球(走盘)",因突尼斯低位会压净胜球
  · 爆冷概率:约 20%——靠突尼斯铁桶 + 门将神勇 / 日本核心缺阵
```

This example demonstrates **format and reasoning style**, not a live call — at run time the
handicap and goals lines come from Channel G's live search, and the picks follow the
concentration of the actual posterior.
