# Output Format (输出格式)

Produce the report **in Chinese**, in exactly these three blocks, in this order. Keep the
reasoning visible — every pick carries its own sub-analysis, and nothing is asserted that the
evidence doesn't support.

## The template (authoritative)

```
一、整体方向分析
<2–4 段你的总体研判>。明确点出支撑判断的关键信息源,而不是空谈:
  · 评级/实力差(Channel A)
  · 近况与底层数据 xG(Channel D)
  · 阵容/伤病/停赛(Channel F)——区分"已确认"与"预计"
  · 对位与默契(Channel C/E)——天赋天花板要扣默契折扣
  · 盘口隐含概率(Channel G)——以及我和市场分歧在哪
给出方向性结论:谁占优、占多大优、平局风险多高、比赛大体走向(开放对攻 / 闷 / 一边倒)。

二、分项预测(每项附"子分析",比整体更细,且必须可解释)

【胜平负】(1–2 个)
  预测:____
  子分析:从比分矩阵汇总得到的 H/平/A 概率 + 主要驱动机制 + 为什么是这个而非相邻选项。

【让球胜负】(1–2 个;让球线先联网查到)
  当前盘口:____ 让 ____ 球(联网确认;查不到则注明"线未确认,定性判断")
  预测:____ 方过盘
  子分析:把让球线套到比分矩阵上得到的过盘概率 + 半/全输赢(若为半球/一球半盘)说明。

【比分】(2–3 个;置信度越高个数越少)
  预测:____ / ____ (/ ____)
  子分析:矩阵概率最高的几个比分;若头部比分明显领先→给 2 个;若多个比分接近→给 3 个。

【进球数 / 大小球】(2 个;大小球线先联网查到)
  当前盘口:大小 ____ 球(联网确认)
  预测:____ / ____
  子分析:矩阵按总进球求和与盘口比较 + 两队 xG 与比赛状态(必胜则开放/无关紧要则保守)。

【半全场】(1–2 个;方差最大,通常 1 个)
  记法:[半场][全场],主队(列前队)视角,各取 胜/平/负,共 9 种(胜胜/胜平/胜负/平胜/平平/
        平负/负胜/负平/负负);负胜=逆转、胜负=被翻盘,概率最低。
  预测:____ (/ ____)
  子分析:基于上下半场各自进球速率的联合判断(别只把全场结果一拆为二);坦白说明这是最不稳的一项。

三、预测总结
  一眼看完的结论清单,直接排布(见下方示例的"总结"块)。
```

## 置信度 → 个数(硬规则,不是装饰)

- 个数由**分布的集中度**决定,不是凭感觉:某类玩法分布越尖,个数越少。
  - 比分:头部比分概率明显领先 → 2 个;多个比分挤在一起 → 3 个。
  - 胜平负:某一结果主导 → 1 个;接近 → 给 2 个(或一个含平的双选)。
  - 半全场:方差最大 → 通常 1 个,只有上下半场都高度同向时给 2 个。
- 每个预测打一个 **置信度标签(高/中/低)**,低置信就如实说,不要凑数。

## 锦上添花(建议加上)

- 每个预测后缀 `置信度:高/中/低`。
- 一个 **关键变量(swing factors)** 小块:最可能改变研判的 2–3 件事(如某核心是否首发、开场是否丢球)。
- 一行 **盘口 vs 我的判断**:我的估计与市场隐含概率分歧最大的点,以及我凭什么这么看。

## 示例(基于真实赛前背景;比分/盘口数值为演示,实战需联网确认)

> 演示赛事:2026 世界杯 F 组,突尼斯 vs 日本(6/21,蒙特雷)。真实背景:首轮突尼斯 1–5 负瑞典、
> 防线崩盘,近 5 场 1 胜 1 平 3 负、进 2 失 11;日本 2–2 战平荷兰、卡马达(Kamada)88 分钟绝平,
> 创造力核心久保(Kubo)/堂安(Doan)/卡马达,中场田中(Tanaka)+佐野(Sano)双后腰。两队均需分数。
> **以下让球线/大小球线为示例占位,真实运行时由 Channel G 联网取实时盘口。**

```
一、整体方向分析
日本占据明显但非压倒性的优势。实力评级、底层数据、对位三条线一致指向日本:首轮对荷兰能踢出
2–2 且 xG 不落下风,说明其控球—边路推进体系成型;久保、堂安、卡马达的创造力对突尼斯刚刚被瑞典
打穿的防线是直接威胁(突尼斯首轮失 5 球、近 5 场失 11,防守组织是结构性问题而非偶然)。突尼斯
的现实指望是低位密集 + 定位球(唯一进球来自雷基克头槌),但这需要中场顶住日本的控球节奏,而这恰
是他们最吃紧的地方。默契与状态上日本更稳(主力框架延续),突尼斯刚换帅压力下需要重组。
分歧提示:盘口若给日本较深的让步,需警惕——世界杯单场方差大,突尼斯摆大巴 + 日本临门效率波动
会压低日本的过盘概率,因此我对"让球"比对"胜平负"更谨慎。
整体走向:日本控球主导、突尼斯被动防反 + 拼定位球;日本进球概率高,但若久保/卡马达任一不首发,
威胁度明显下降。

二、分项预测

【胜平负】(1 个) 预测:日本胜  置信度:中高
  子分析:矩阵汇总日本胜约六成、平约两成出头、突尼斯胜两成以内。驱动:体系与创造力差 + 突尼斯
  防守结构性漏洞。不取双选是因为头部明显偏向日本;但不上调到"大胜"级别,因突尼斯低位防守会压低
  净胜球。

【让球胜负】(1 个)
  当前盘口:日本 让 1 球(示例,需联网确认)
  预测:倾向突尼斯 +1 受让(不过盘/打出受让方)  置信度:中
  子分析:把"日本 -1"套到比分矩阵,日本赢 2 球及以上才赢盘,而突尼斯摆大巴会把比赛压成小比分,
  1–0 这类最可能的赢法只能走盘/输盘。若实盘是 -0.5/-1,则日本赢盘概率回升——所以**线必须先查到**。

【比分】(3 个) 预测:1–0 日本 / 2–0 日本 / 2–1 日本  置信度:中
  子分析:给 3 个是因为头部比分较分散——突尼斯低位让"日本小胜"成为一簇接近的结果,没有单一比分
  显著领先,故不收敛到 2 个。

【进球数 / 大小球】(2 个)
  当前盘口:大小 2.5 球(示例,需联网确认)
  预测:小于 2.5 球(主) / 总进球 1–2 球(次)  置信度:中
  子分析:日本进攻强但突尼斯主动求稳、压缩空间,且突尼斯自身进攻贫弱(近 5 场进 2),总进球期望
  被压低,矩阵质量集中在 1–2 球区间。若突尼斯被迫追分(下半场落后)则上调,故与比赛状态强相关。

【半全场】(1 个) 预测:平负(半场平、全场日本胜;主队突尼斯视角)  置信度:低
  子分析:突尼斯开场通常先求稳,日本破密集防守往往需要时间,上半场 0–0、下半场日本打开的路径
  概率最高。坦白:半全场方差最大,仅作低权重参考。

三、预测总结
  · 胜平负:日本胜(中高)
  · 让球(日本 -1 示例):突尼斯 +1 受让/不过盘(中)——以实时盘口为准
  · 比分:1–0 / 2–0 / 2–1 日本(中)
  · 大小球(2.5 示例):小球 / 1–2 球(中)
  · 半全场:平负(半场平、全场日本胜)(低)
  · 关键变量:久保&卡马达是否首发;突尼斯是否仍摆大巴;日本是否早进球打开局面
  · 盘口 vs 我:让球盘我比市场更偏向受让方(突尼斯低位会压净胜球)
```

This example demonstrates **format and reasoning style**, not a live call — at run time the
handicap and goals lines come from Channel G's live search, and the picks follow the
concentration of the actual posterior.
